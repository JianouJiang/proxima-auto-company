# PowerCast User Feedback Log

**记录所有真实用户反馈，寻找模式。**

---

## 反馈分类

每条反馈标记为以下类别之一：

- **[BUG]** — 产品有问题（坏掉的链接、错误的数据、无法访问）
- **[FEATURE]** — 用户想要的新功能
- **[CONFUSION]** — 用户不理解某个东西（可用性问题）
- **[PRAISE]** — 用户说"这个好"
- **[CRITICISM]** — 用户说"这个不行"
- **[QUESTION]** — 用户问问题（不是抱怨，只是好奇）
- **[NEED]** — 用户描述了真实需求（即使没有明确要求功能）

---

## 反馈记录格式

```
### [日期] — [来源] — [分类]

**反馈内容：**
[用户说了什么]

**用户背景：**
[这个人是谁？能源交易员？储能运营商？研究员？]

**我的回应：**
[你回复了什么]

**Action Item：**
[需要做什么？修复 bug？加功能？改文案？]

**优先级：**
[High / Medium / Low]

---
```
